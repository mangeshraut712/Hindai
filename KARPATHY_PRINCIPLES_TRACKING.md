# Karpathy Principles Tracking

**Last Updated:** April 24, 2026

## The Four Core Principles

### 1. Think Before Coding (Ambiguity Management)

- Never make assumptions about ambiguous requirements
- Stop and ask for clarification if unclear
- Surface technical tradeoffs before implementation
- Do not silently pick an interpretation

### 2. Simplicity First (Minimalism)

- Prioritize minimum code required
- Avoid speculative abstractions
- No over-engineering or unnecessary flexibility
- If 50 lines can solve it, don't write 200

### 3. Surgical Changes (Scope Control)

- Modify only specific code required
- Do not refactor neighboring code
- Do not delete comments unless instructed
- Maintain existing codebase integrity

### 4. Goal-Driven Execution (Measurable Targets)

- Transform vague instructions into verifiable targets
- Convert qualitative to quantitative objectives
- Define success criteria before coding
- Loop until verified
